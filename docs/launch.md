# Launch Notes

## Chosen names

- Organization: `contextbeacon-labs`
- Repository: `contextbeacon`

## Why these names

- `ContextBeacon` is readable and matches the product.
- The word `beacon` suggests guidance and visibility for agents.
- The repo name is short enough for GitHub and CLI usage.

## README shape

1. Title and one-line pitch
2. Why this exists
3. What it will do
4. First release structure
5. Suggested stack
6. Roadmap
7. Contributing
8. License

## Public launch checklist

- Create the GitHub organization
- Push the repository as public
- Add a MIT license
- Add the first issue list for scanner, generator, and linter work
